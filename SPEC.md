## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 新創團隊（5-20 人） | ~5,000 | 對外介紹團隊需美觀但無設計資源 |
| 人資部門 | ~3,000 | 招募頁需要團隊介紹、製作耗時 |
| 行銷公司 | ~2,000 | 客戶要團隊頁但每個客戶都要重做 |
| 個人工作者 / 接案者 | ~10 萬 | 想打造個人品牌但無設計能力 |

### 1.2 為什麼不做替代方案

- **商用團隊介紹工具（About.me）**：單頁式、無法深度客製
- **LinkedIn 公司頁**：介面固定、無法客製視覺
- **WordPress / Wix 自架**：需設計 + 維護成本
- **我們的解法**：輸入團隊成員資料 → AI 自動生成格式化介紹文案 + 視覺化頁面 + 一鍵發布，純前端

---

## 2. 解決方案

### 2.1 核心價值主張

> 「輸入成員資料 → AI 生成介紹頁 → 一鍵發布 — 5 分鐘搞定專業團隊頁。」

### 2.2 使用者流程

1. 輸入團隊成員資料（姓名/職稱/背景/技能/照片）
2. AI 自動生成每人的介紹文案（3 種風格：專業/親切/創意）
3. 選擇頁面模板（3 種設計風格）
4. 預覽 + 調整
5. 一鍵匯出（靜態 HTML/Markdown/Notion 子頁）

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 團隊成員 CRUD（姓名/職稱/背景/技能/照片/Email）
- [ ] AI 自動生成介紹文案（3 種風格）
- [ ] 3 種頁面模板（簡約/卡片/雜誌）
- [ ] 預覽模式（桌機/手機）
- [ ] 匯出靜態 HTML
- [ ] 匯出 Markdown
- [ ] Notion 子頁同步

### 3.2 v2（加值）

- [ ] 多人協作編輯
- [ ] 動態生成（依受眾切換文案）
- [ ] SEO 優化
- [ ] 團隊值日輪播

### 3.3 明確不做

- 自訂網域託管（純產生頁面）
- 招募流程管理
- 員工績效評估
- 多語系介面

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| LLM 產出文案品質 | 🟠 中 | 使用者可手動修改 + 重新生成 |
| 個資疑慮（員工資料） | 🟡 低 | 加密儲存 + 隱私頁聲明 |
| 照片肖像權 | 🟡 低 | 明確聲明使用者需取得員工同意 |
| 模板設計 | 🟡 低 | 參考業界標準設計 |
